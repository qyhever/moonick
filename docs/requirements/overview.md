## 全局 / 公共规范

### 1. 认证与鉴权

- H5 业务端采用 JWT 方案。
- `accessToken` 有效期为 **2 小时**，`refreshToken` 有效期为 **72 小时**。
- H5 用户 token 存储在 **localStorage**，关闭 H5 后登录状态保留。
- access token 过期后，使用 refresh token 无感续签，并恢复原 401 请求。
- H5 用户体系与 PC 管理端管理员体系**分离**，鉴权不得混用。

### 2. 分页规范

- 默认采用 offset 分页。
- 分页参数统一为：`pageNum` + `pageSize`。
- 默认 `pageSize = 10`。

### 3. 行程状态枚举

| 状态值 | 含义 |
|--------|------|
| `active` | 正常可报名 |
| `full` | 已满员 |
| `closed` | 手动关闭 |
| `expired` | 已过期（服务端定时任务触发） |

### 4. 用户信息字段

| 字段 | 规范 |
|------|------|
| 昵称 | `nickname` |
| 头像 | 返回完整 URL；有系统默认头像 |

### 5. 网络错误处理

| 错误类型 | 前端提示文案 |
|----------|-------------|
| 请求超时 | 「请求超时，请稍后重试」 |
| 网络断开 | 「网络连接已断开，请检查网络」 |
| 服务端 5xx | 「服务器异常，请稍后再试」 |

## 版本边界

当前 v1 版本以轻量为主，优先保证核心流程顺畅：

- H5 端聚焦：浏览、发布、收藏、登录注册、个人中心。
- PC 端聚焦：管理登录、基础看板、行程管理、用户查询。

以下能力不纳入当前 v1 范围：

- 支付功能
- 地图集成
- 评价体系
- 复杂 BI 图表
- 多角色权限（RBAC）
- 工单、投诉、营销配置
