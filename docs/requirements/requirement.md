# 明叶同行 — 产品需求文档

---

## 产品概览

| 项目 | 内容 |
|------|------|
| 产品名 | 明叶同行 |
| 产品类型 | 拼车 H5 移动端应用 |
| 目标用户 | 车主（发布车找人）、乘客（发布人找车） |
| 核心功能 | 行程发布、行程浏览、收藏管理 |
| 核心流程 | 发布 → 浏览 → 联系 |

---

## 全局 / 公共规范

### 视觉 & 交互

- **视觉风格**：简约、干净，白底 + 主色调（深绿/蓝绿系，体现出行感），圆角卡片，轻投影
- **底部 Tabbar**：固定 3 Tab（首页｜我要发布｜我的），图标 + 文字，当前 Tab 高亮
- **品牌语**：`诚信拼车 · 安全同行`，全局页脚展示
- **加载态**：
  - 首次进入页面：使用**骨架屏**
  - 下拉刷新：使用**顶部 loading 指示器**，不再触发骨架屏
- **空状态**：
  - 列表仅一页且条目较少：展示插画 + **「暂无数据」**
  - 上拉加载到底：底部展示**「没有更多了」**
- **Toast 规范**：统一展示 **2 秒**（错误类型和成功类型均适用），**居中**显示
- **弹窗关闭规则**：点击背景蒙层**不能**关闭弹窗，必须通过操作按钮关闭

### 认证 & 鉴权

- **方案**：JWT，`accessToken` 有效期 **2 小时**，`refreshToken` 有效期 **72 小时**
- **存储**：token 存储在 **localStorage**，关闭 H5 后登录状态保留
- **续签策略**：不支持无感续签；token 过期后**直接跳转登录页**
- **登录入口**：统一使用**登录页**，不使用登录弹窗
- **回跳机制**：登录成功后通过 **query param（`redirect`）** 回跳来源页并执行对应操作
- **多设备**：不限制，允许同一账号多设备同时登录

### 网络错误处理

| 错误类型 | 前端提示文案 |
|----------|-------------|
| 请求超时 | 「请求超时，请稍后重试」 |
| 网络断开 | 「网络连接已断开，请检查网络」 |
| 服务端 5xx | 「服务器异常，请稍后再试」 |

## 一、首页（行程列表）

### 1.1 筛选栏

| 筛选项 | 组件类型 | 说明 |
|--------|----------|------|
| 类型 | 单选 chip | 全部 / 车找人 / 人找车（"全部"为默认项） |
| 起点 | **自由文本输入框** | 防抖延迟 **300ms** 后触发搜索/筛选 |
| 终点 | **自由文本输入框** | 防抖延迟 **300ms** 后触发搜索/筛选 |
| 时间 | 单选 chip | 全部 / 今天 / 明天 |
| 重置 | 按钮 | 一键清空所有筛选条件 |

### 1.2 行程卡片

| 字段 | 展示规则 |
|------|----------|
| 类型标签 | 车找人（橙色）/ 人找车（蓝色），色差区分 |
| 行程路线 | 【出发地】→【目的地】，加箭头图标，超长截断 |
| 出发时间 | X 月 X 日 XX:XX 出发；今天/明天出发时额外标注"今天""明天" |
| 剩余座位 | 剩余 X 个座位；0 座时展示红色「已满」标签，卡片禁止点击 |
| 人均费用 | ¥XX；面议用灰色斜体展示 |
| 收藏图标 | 右上角心形图标；**已收藏：实心红色心形**；切换时触发**弹跳动画**；操作无 Toast 反馈；未登录时点击跳转登录页 |
| 发布时间 | 卡片底部展示「X 小时前发布」 |

### 1.3 列表交互

- **下拉刷新**：顶部 loading 指示器
- **上拉加载更多**（分页）：参数 `pageNum` + `pageSize=10`（offset 分页）
- **列表到底**：底部显示「没有更多了」
- **无数据**：插画 + 「暂无行程，去发一个吧」引导文案（显示「暂无数据」）

## 二、发布行程页

> ⚠️ **需要登录**才能进入此页；未登录用户点击「我要发布」Tab 时，跳转登录页，登录成功后自动回跳

### 2.1 表单项

| 字段 | 组件类型 | 规则说明 |
|------|----------|----------|
| 行程类型 | 大按钮切换组（车找人 / 人找车） | 必填，突出选择感 |
| 起点 | **自由文本输入框** | 必填 |
| 终点 | **自由文本输入框** | 必填 |
| 出发日期 | 日历组件 | 必填，只可选今天及以后 |
| 出发时间 | 时间选择器 | 必填，与日期分开选择 |
| 人数 | 数字步进器（+/-） | 必填，范围 1～6 人 |
| 人均费用 | 金额输入框 + 「面议」复选（二选一） | 必填 |
| 微信号 | 文本输入框 | 选填（与手机号至少填一项） |
| 手机号 | 数字输入框 | 选填（11 位手机号格式校验，与微信号至少填一项） |
| 备注 | **自由文本输入框**（placeholder 提示示例内容） | 选填 |

### 2.2 前端校验规则

| 校验项 | 触发时机 | 提示文案 |
|--------|----------|----------|
| 起点 = 终点 | 点击发布时 | 「起点和终点不能相同」 |
| 出发时间早于当前时间 | 点击发布时 | 「出发时间不能早于当前时间」 |
| 微信号与手机号均未填写 | 点击发布时 | 「请填写至少一种联系方式」 |

### 2.3 发布逻辑

1. 点击发布按钮 → **统一执行表单校验**
2. 校验通过 → **直接提交**（无确认弹窗，无预览步骤）
3. 发布成功 → 跳转至该行程详情页，顶部 Toast 提示「发布成功」
4. 发布失败 → Toast 提示具体错误信息

## 三、行程详情页

### 3.1 展示字段

| 字段 | 展示说明 |
|------|----------|
| 类型标签 | 同列表样式（色差区分） |
| 行程路线 | 出发地 → 目的地，大号字体突出展示 |
| 出发时间 | 日期 + 时间完整展示 |
| 剩余座位 | 含「已满」状态 |
| 人均费用 | ¥XX / 面议 |
| 微信号 | 展示 + 点击可一键复制，复制后 Toast「复制成功」 |
| 手机号 | 展示 + 点击**直接调用系统拨号**（无确认弹窗） |
| 备注 | 完整展示所有备注内容 |
| 发布时间 | 「X 天前发布」 |

### 3.2 行程已满员状态

当行程座位为 0（`status: full`）时：
- 页面**顶部展示 Banner**：「该行程已满员」
- 底部操作栏**收藏和联系按钮禁用**（置灰不可点击）

### 3.3 底部操作栏

**他人发布的行程：**
- `❤️ 收藏`（已收藏显示实心红色心形，切换有弹跳动画，无 Toast；未登录跳转登录页）
- `📞 联系TA`（直接系统拨号）

**本人发布的行程：**
- `编辑` + `删除` 两个并排按钮

## 四、个人中心页

> ⚠️ **需要登录**才能进入此页；未登录时展示默认状态，点击菜单项跳转登录页

### 4.1 头部区域

- **未登录状态**：展示默认头像 + 「点击登录」引导，点击跳转登录页
- **已登录状态**：展示头像（点击可更换）+ 昵称（字段名：`nickname`，点击可编辑）

**头像更换规则：**
- 支持格式：`jpg / png / webp / heic`，最大 **10MB**
- **无裁剪步骤**，上传后**立即本地预览**（不等待接口返回）
- 若接口返回上传失败，**回退至原头像预览**
- 头像字段返回**完整 URL**，有**系统默认头像**

### 4.2 功能菜单

| 菜单项 | 说明 |
|--------|------|
| 我的发布 | 展示发布数量角标（超过 99 显示「**99+**」），进入我的发布列表 |
| 我的收藏 | 展示收藏数量角标（超过 99 显示「**99+**」），进入我的收藏列表 |
| 联系方式管理 | 修改常用微信号/手机号 |
| 关于我们 | 产品简介 + 版本号 |

### 4.3 我的发布列表 / 我的收藏列表

- 卡片样式同首页行程卡片
- 我的发布支持：手动关闭行程（已满员/取消行程）；行程状态枚举：`active / full / closed / expired`
- **收藏联动**：行程被删除后，收藏记录**保留**；用户点击跳转时返回「行程不存在」提示
- 支持下拉刷新、上拉加载更多；分页参数同首页（`pageNum + pageSize=10`）

## 五、登录 / 注册页

### 5.1 登录页

| 字段 | 说明 |
|------|------|
| 手机号 | 数字输入框，11 位校验 |
| 密码 | 密码输入框，支持明文切换 |
| 登录按钮 | 校验通过后提交 |
| 去注册 | 跳转注册页 |

- 登录成功后，通过 `redirect` query param 回跳来源页并执行对应操作
- token 过期后无感续签**不支持**，直接跳至此页（无需恢复之前的操作/表单）

### 5.2 注册页

| 字段 | 说明 |
|------|------|
| 手机号 | 数字输入框，11 位校验，需唯一 |
| 密码 | 密码输入框，6～20 位 |
| 确认密码 | 与密码一致性校验 |
| 注册按钮 | 校验通过后提交，注册成功自动登录并回跳来源页 |
| 去登录 | 跳转登录页 |

**错误提示：**
- 手机号已注册：**Toast 弹出提示**「该手机号已注册，请直接登录」（不使用行内红字或弹窗）

## 六、技术规范摘要

> 本节汇总前后端对接的关键规范，供开发直接参考。

### 认证

| 项目 | 规范 |
|------|------|
| 方案 | JWT |
| accessToken 有效期 | 2 小时 |
| refreshToken 有效期 | 72 小时 |
| 存储位置 | localStorage |
| 续签策略 | 不支持，过期直接跳转登录页 |
| 多设备限制 | 无限制 |

### 分页

| 项目 | 规范 |
|------|------|
| 分页方式 | offset 分页 |
| 请求参数 | `pageNum`（页码）+ `pageSize`（默认 10） |

### 行程状态枚举

| 状态值 | 含义 |
|--------|------|
| `active` | 正常可报名 |
| `full` | 已满员 |
| `closed` | 车主手动关闭 |
| `expired` | 已过期（服务端定时任务触发） |

### 用户信息字段

| 字段 | 规范 |
|------|------|
| 昵称 | `nickname` |
| 头像 | 返回完整 URL；有系统默认头像 |

### 收藏接口

- 使用**幂等 toggle 端点**，同一接口处理收藏和取消收藏

## MVP 范围说明

当前 v1 版本以轻量为主，优先保证核心流程顺畅：

```
[游客] 浏览行程列表 → 查看详情 → 联系对方（无需登录）
                                      ↓ 点击收藏 / 发布
                               跳转登录页（query param 回跳）
                                      ↓
                        登录 / 注册（手机号 + 密码）
                                      ↓
                  发布行程 / 收藏行程 / 管理个人中心
```

评价体系、支付功能、地图集成等复杂功能放入后续迭代版本。
