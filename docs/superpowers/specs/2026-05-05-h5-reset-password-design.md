# H5 重置密码页面同步设计

日期：2026-05-05

## 背景

`htmls/v1.html` 中已经完成“重置密码”静态稿确认，当前 `mn-frontend-h5` 仅在账户设置页保留了“账号安全”占位项，尚未同步对应的独立 React 页面。

本次目标是将 HTML 静态稿中的“重置密码”部分同步到 H5 React 代码中，仅覆盖页面结构、文案、样式与前端校验，不接入真实后端密码重置接口。

## 范围

纳入本次范围：

- 在账户设置页的“安全管理”区域新增明确的“重置密码”入口
- 新增独立 H5 路由页承载重置密码内容
- 同步 `htmls/v1.html` 的核心信息层级、文案与视觉结构
- 实现前端表单校验
- 点击提交后展示“暂未接入”提示，停留当前页
- 补充对应测试

不纳入本次范围：

- 不调用后端接口
- 不处理短信验证码、邮件验证码或当前密码校验
- 不修改登录、注册、找回密码等鉴权流程
- 不扩展设备管理、风控管理等安全能力

## 目标页面

### 1. 账户设置页

位置：`/me/settings`

“安全管理”区块中的现有占位项从“账号安全”调整为“重置密码”入口，文案与 HTML 静态稿保持一致：

- 标题：`重置密码`
- 描述：`定期更新登录密码，保护账户登录安全`
- 操作文案：`去修改`

点击后跳转到独立路由页，不在当前页内展开表单。

### 2. 重置密码页

建议路由：`/me/settings/password-reset`

页面按静态稿同步为四层结构：

1. 顶部返回栏
2. Hero 说明卡
3. 表单卡
4. 安全建议区

## 页面结构设计

### 顶部返回栏

- 左侧返回按钮
- 中间标题：`重置密码`
- 右侧为空白占位，保持标题居中

返回行为使用前端路由导航回到账户设置页。

### Hero 说明卡

保留静态稿中的定位与核心说明：

- eyebrow：`Password Reset`
- 主标题：`请设置新的登录密码，让账户安全保持在你手里。`

这部分的目的不是增加功能，而是明确告诉用户当前是“重置密码”而非“修改密码”场景。

### 表单卡

表单字段：

- `新密码`
- `确认密码`

表单说明：

- 标题：`重置密码`
- 副标题：`输入新的登录密码，并再次确认，避免因输入错误影响后续登录。`

按钮：

- 主按钮文案：`确认`

安全提示区：

- 标题：`安全提示`
- 内容：`密码建议包含字母与数字组合，并避免与其他平台使用相同密码。`

### 安全建议区

保留静态稿的信息层级，展示两条建议卡片：

- `不要重复使用旧密码`
- `避免与其他平台共用密码`

该区域为静态说明，不参与表单逻辑。

## 交互设计

### 表单输入

- 两个字段均为密码类型输入框
- 均支持受控输入
- 默认空值

### 校验规则

提交时执行以下前端校验：

1. 新密码不能为空
2. 确认密码不能为空
3. 两次输入必须一致

本次不引入更强的密码强度规则，避免前端自行定义与未来后端规则冲突。

### 提交行为

当校验通过后：

- 不发起 API 请求
- 展示静态提示：`暂未接入`
- 停留在当前页面

如果校验失败：

- 在页内显示对应错误提示
- 不提交

## 样式策略

实现原则：

- 以 `mn-frontend-h5/src/styles/h5.css` 的现有视觉语言为基础
- 补充一组聚焦于重置密码页的样式类，如 `password-reset-*`
- 尽量复用现有 `card`、表单、标题、按钮、壳层风格

样式目标：

- 尽量还原 `htmls/v1.html` 中已经确认过的视觉层级
- 保持与当前 H5 项目整体风格一致
- 不新引入外部 UI 库或新的页面设计体系

## 路由与组件边界

建议拆分如下：

- 账户设置页继续保留在现有文件中，只负责入口展示
- 新增独立页面组件，例如 `PasswordResetPage`
- 在路由表中新增受保护路由

这样可以保持边界清晰：

- `AccountSettingsPage` 负责资料与安全入口聚合
- `PasswordResetPage` 负责单一的重置密码视图与前端校验

## 测试设计

至少覆盖以下场景：

1. 账户设置页显示“重置密码”入口
2. 点击入口可进入重置密码页
3. 重置密码页显示关键标题与说明文案
4. 空表单提交时出现错误提示
5. 两次密码不一致时出现错误提示
6. 校验通过后显示“暂未接入”提示

本次不测试网络请求，因为该页面不接后端接口。

## 风险与约束

- 当前只是静态前端同步，后续如果后端接口字段或流程确定，提交逻辑需要补接
- 若未来产品要求“找回密码”与“登录后重置密码”区分流程，本页面文案和交互需再次校准
- 当前密码强度规则故意保持最小化，避免形成错误的前端事实标准

## 实施建议

推荐采用以下实施顺序：

1. 调整账户设置页安全管理入口文案与跳转
2. 新增受保护路由与重置密码页面组件
3. 同步页面结构与样式
4. 补前端校验与“暂未接入”提示
5. 补测试并执行验证
