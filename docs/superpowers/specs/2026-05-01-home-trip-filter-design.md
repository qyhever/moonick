# 首页行程列表筛选改造设计

## 背景

`htmls/v1.html` 的首页当前以品牌展示和快捷入口为主，行程列表缺少条件查询，不利于 H5 用户快速找车找人。

## 决策

首页保持“首页即行程列表”的结构，不拆分独立列表页。

## 设计方案

### 首页定位

- 首页继续承载品牌露出、快捷入口和行程列表。
- 首页首屏新增精简筛选区，直接服务“快速找行程”。

### 精简筛选

- 首页默认展示 3 个高频条件：
  - 类型：全部 / 车找人 / 人找车
  - 起点：自由文本输入
  - 终点：自由文本输入
- 这部分始终在列表上方可见，作为主筛选入口。

### 筛选抽屉

- 将低频但必要的操作收纳到“筛选抽屉”：
  - 时间：全部 / 今天 / 明天
  - 重置
  - 更多条件入口占位
- 抽屉采用底部弹出层表达，符合 H5 常见交互。
- 当前原型阶段以静态展开示意表达，不引入完整交互逻辑。

### 列表呈现

- 列表标题从“附近行程”调整为“行程列表”，弱化内容推荐感，强化检索感。
- 标题旁补充结果说明，如“按起终点快速筛选”。
- 保留现有卡片风格，只增强首页信息架构，不改卡片主视觉。

## 取舍说明

- 不拆独立列表页：避免多一次点击，保持首页即核心浏览页。
- 不把所有条件平铺首页：避免首屏过重，影响浏览效率。
- 不在本次原型中加入排序、历史搜索、热门路线：超出 v1 必要范围。

## 验收要点

- 首页能明显看到筛选入口。
- 时间和重置不占首页主空间，而是进入抽屉。
- 首页仍然以行程列表为主体，而不是退化成门户页。
