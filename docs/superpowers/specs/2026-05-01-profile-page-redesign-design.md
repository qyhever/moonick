# `htmls/v1.html` “我的”页面重设计规格

日期：2026-05-01

## 目标

重设计 `htmls/v1.html` 中的“我的”页面，解决当前单张卡片内信息堆叠、视觉层级混乱、危险操作与资料编辑混排的问题。

本次仅调整静态 HTML 页面中的该页结构与样式，不改动其他页面的信息架构，不新增后端交互能力。

## 现状问题

- 头像、昵称、手机号、统计数据、资料编辑输入框、保存按钮、退出按钮全部堆在同一张卡中。
- 主要信息与次要操作没有分层，阅读路径不明确。
- “退出登录”与“保存修改”并列放在表单区内，语义冲突明显。
- 页面虽然延续整体暖色风格，但“我的”页缺少独立的身份感和账户中心感。

## 设计方向

采用已确认的 `A. 暖调杂志感` 方向：

- 延续当前页面已有的暖色、奶油底、柔和阴影视觉语言。
- 用更强的顶部身份头图承载用户身份信息和关键数据。
- 用多张白色卡片拆分功能分区，形成清晰上下阅读流。
- 保持移动端轻量、可扫读、易点击，不引入复杂交互。

## 页面结构

页面分为 4 个区块，自上而下排列：

1. 身份头图卡
2. 常用服务卡
3. 账户设置卡
4. 底部操作区

### 1. 身份头图卡

用于承载最重要的身份信息和账号状态：

- 头像首字母
- 昵称
- 脱敏手机号
- 实名状态标签
- 关键统计数据：`我的发布`、`我的收藏`、`信用评分`

视觉要求：

- 使用深色暖调渐变背景，与当前全站暖色体系一致但层级更高。
- 保留柔和发光装饰，避免过于平。
- 统计项以内嵌小卡形式展示，减少散乱感。

### 2. 常用服务卡

用于承载高频快捷入口，采用 4 宫格：

- 证件信息
- 钱包余额
- 常用乘客
- 客服帮助

视觉要求：

- 白底卡片，与头图形成层次对比。
- 每个入口使用独立淡色图形底块，延续暖调但区分功能。
- 标题与说明文案独立，不与数据区混排。

### 3. 账户设置卡

用于展示和编辑账户相关信息，采用列表式信息项：

- 昵称
- 默认手机号
- 账号安全

交互表达：

- 采用“标签 + 当前值 / 去管理”的列表排布。
- 不在首屏直接暴露多个输入框，避免界面显得像原始表单。
- 保留“可修改”的语义，但视觉上更像账户中心而不是信息录入页。

## 4. 底部操作区

底部操作独立于信息卡片：

- 主操作：`保存修改`
- 次操作 / 危险操作：`退出登录`

要求：

- 两个按钮并排但风格区分明显。
- `退出登录` 不再混在资料信息区中。

## 风格与实现约束

- 保持 `htmls/v1.html` 现有整体设计语言，不重做全文件主题。
- 优先复用现有字体、阴影、圆角和暖色变量。
- 新增样式应尽量围绕“我的”页局部 class，避免影响首页、发布页、详情页等已有内容。
- HTML 结构应语义清晰，便于后续替换成真实组件或 React 页面。

## 不在本次范围

- 不新增真实跳转逻辑
- 不新增头像上传、表单校验、设置页详情展开
- 不重构底部 Tab Bar
- 不修改非“我的”页面布局

## 验收标准

- 用户在首屏能快速区分“身份信息 / 数据概览 / 常用服务 / 账户设置 / 退出登录”这 5 类内容。
- 页面不再出现单卡堆叠全部信息的混乱观感。
- “我的”页与当前站点暖色风格保持一致，但整体层级明显更成熟。
- 仅浏览静态页面时即可感知结构优化，无需依赖交互解释。
