# H5 首页筛选收敛设计

日期：2026-05-01
范围：`mn-frontend-h5` 首页筛选区

## 目标

将首页首屏筛选收敛为高频操作，降低主卡片复杂度。

本次调整后：

- 主区只保留 `车找人`、`人找车` 两个类型切换
- 其余筛选条件统一收进底部抽屉
- 在类型切换同一行右侧新增 `LayoutGrid` 图标作为抽屉入口

## 交互设计

### 主筛选区

- 删除主区中的 `全部` 类型入口
- 删除主区中的 `起点`、`终点` 输入框
- 删除主区中的 `查询` 按钮
- 删除现有整行 `更多筛选` 触发器
- 保留两枚类型 chip：
  - `车找人` -> `driver_post`
  - `人找车` -> `passenger_post`
- 右侧新增单独图标按钮，点击后打开筛选抽屉

### 抽屉区

抽屉承载以下全部非类型筛选：

- 起点 `fromText`
- 终点 `toText`
- 时间 `datePreset`
- 仅看可约 `onlyAvailable`
- 可议价 `negotiableOnly`

抽屉底部保留：

- `重置`
- `确定`

行为约定：

- 打开抽屉时，将当前已生效筛选同步到抽屉草稿态
- 点击 `确定` 后一次性提交抽屉内全部条件
- 点击 `重置` 仅重置抽屉内条件，不影响主区类型
- 点击遮罩或关闭按钮关闭抽屉，不提交未确认修改

## 数据与请求

- `tripType` 继续即时生效
- `fromText`、`toText` 不再在主区 debounce 查询，改为抽屉确认后统一提交
- 列表请求仍通过现有 `buildTripQuery` 生成
- `onlyAvailable`、`negotiableOnly` 继续保留为前端本地过滤

## 视觉约束

- 图标使用 `lucide-react` 的 `LayoutGrid`
- 图标按钮与类型 chip 位于同一行，靠右对齐
- 样式延续现有首页暖色体系，不引入新的视觉系统
- 图标按钮应具备明确点击热区，不做纯装饰展示

## 测试影响

需要同步更新首页测试：

- 旧的主区起点/终点输入与查询按钮测试不再成立
- 新增或调整抽屉内起点/终点筛选提交流程测试
- 保留类型切换、抽屉时间筛选、抽屉重置/关闭行为测试

## 非目标

- 不改动后端接口契约
- 不新增新的筛选字段
- 不调整行程列表卡片内容
