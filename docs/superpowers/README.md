# superpowers 过程归档

本目录用于保存历史计划、规格草案、暂停摘要、阶段总结和过程性材料。

## 使用原则

- 这里的文档用于追溯“当时怎么想、怎么做、为什么停在这里”。
- 这里的文档默认不是当前规范来源，也不保证始终与代码和正式文档同步。
- 需要查看当前有效规则时，优先阅读：
  - `../../AGENTS.md`
  - `../index.md`
  - `../requirements/*`
  - `../technical/*`

## 子目录说明

- `plans/`：阶段性实现计划
- `specs/`：设计规格与方案草案
- 根目录日期文件：过程总结、暂停记录和阶段收口文档
