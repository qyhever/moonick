## 1. 文档目标

本文档用于指导 `mn-frontend-admin` 的页面实现与后端对接，覆盖后台布局、鉴权、页面功能、通用组件抽象和测试方案。

## 2. 技术选型

| 项目 | 建议 |
|------|------|
| 前端框架 | React |
| 构建工具 | Vite |
| UI 组件库 | Ant Design |
| 路由 | React Router |
| 请求库 | Axios |
| 表单方案 | Ant Design Form |

原则：

- 复用 Ant Design 的后台布局和表单表格能力
- 组件抽象优先围绕“查询表单、分页表格、详情区块、编辑表单”四类基础能力


左侧菜单：

- 首页

- 用户管理

---

## 4. 路由与鉴权

### 4.1 登录态规则

- 管理员登录成功后保存 `accessToken` 和 `refreshToken`
- 未登录访问受保护页面时统一跳转登录页
- access token 过期时自动尝试 refresh
- refresh 失败后跳回登录页

### 4.2 Token 隔离

- Admin 只使用 `/api/admin/v1/...` 接口
- 不允许复用 H5 用户端 token

---

## 5. 页面设计与接口映射

### 5.1 管理员登录页

接口：

- `POST /api/admin/v1/auth/login`

字段：

- 用户名
- 密码

成功行为：

- 保存 token
- 跳转后台首页

### 5.2 首页看板

接口：

- `GET /api/admin/v1/dashboard/summary`

展示内容：

- 行程总数
- 今日新增行程数
- 用户总数
- 当前有效行程数

实现建议：

- 使用 4 张数据卡片
- 提供跳转到行程管理和用户管理的快捷入口

### 5.3 行程列表页

接口：

- `GET /api/admin/v1/trips`

筛选项：

- 行程类型
- 起点
- 终点
- 出发日期
- 行程状态
- 发布人关键字

列表字段：

- 行程 ID
- 行程类型
- 行程路线
- 出发时间
- 座位/人数
- 费用
- 状态
- 发布人
- 发布时间
- 更新时间
- 操作

操作：

- 查看详情
- 进入编辑

### 5.4 行程详情页

接口：

- `GET /api/admin/v1/trips/:id`

展示字段：

- 行程 ID
- 行程类型
- 起点
- 终点
- 出发日期
- 出发时间
- 人数/座位
- 人均费用 / 面议
- 微信号
- 手机号
- 备注
- 行程状态
- 发布时间
- 更新时间
- 发布人信息

### 5.5 行程编辑页

接口：

- `GET /api/admin/v1/trips/:id`
- `PUT /api/admin/v1/trips/:id`

规则：

- 编辑前先加载详情并回填表单
- 保存前统一做表单校验
- 点击保存时必须二次确认
- 保存成功后返回详情页或列表页

推荐校验：

- 起点终点不能相同
- 出发时间不能早于当前时间
- 联系方式至少一项

### 5.6 用户列表页

接口：

- `GET /api/admin/v1/users`

筛选项：

- 昵称
- 手机号
- 注册时间范围

字段：

- 用户 ID
- 头像
- 昵称
- 手机号
- 发布数量
- 收藏数量
- 注册时间
- 操作

### 5.7 用户详情页

接口：

- `GET /api/admin/v1/users/:id`
- `GET /api/admin/v1/users/:id/trips`

展示内容：

- 基本资料
- 数据概览
- 发布行程列表
- 收藏数量或收藏摘要

规则：

- v1 只读，不支持编辑、封禁、删除

## 6. 通用组件建议

建议抽象以下基础组件：

- `AdminLayout`：侧边栏 + 顶栏 + 内容区
- `PageTable`：统一分页表格
- `SearchForm`：统一查询表单
- `DetailSection`：统一详情展示区块
- `ConfirmSubmitButton`：带二次确认的提交按钮

这样可以减少后续后台页面重复实现。

---

## 7. 交互与体验规范

- 列表支持分页与重置筛选
- 空数据展示后台默认空状态
- 保存成功统一使用全局成功提示
- 接口异常统一用消息提示组件反馈
- 禁止出现编辑、封禁、删除用户等超出范围操作

---

## 8. 错误处理

错误处理原则：

- 优先根据后端 `code` 做业务判断
- 未登录统一走 refresh 流程
- refresh 失败统一跳转登录页
- 服务端异常统一提示“服务器异常，请稍后再试”

---

## 9. 测试用例

### 9.1 登录

- 正确账号密码可登录
- 未登录访问后台页面会跳转登录页
- token 过期后自动 refresh

### 9.2 看板

- 看板数据正确加载
- 快捷入口可跳转对应页面

### 9.3 行程管理

- 列表筛选条件正确生效
- 分页切换正确
- 行程详情字段展示完整
- 编辑前弹出二次确认
- 保存成功后列表与详情同步更新

### 9.4 用户管理

- 用户列表支持条件查询
- 用户详情展示发布数量与收藏数量
- 页面中不出现超范围操作按钮

---

## 10. 开发建议

建议实现顺序：

1. 登录页与后台基础布局
2. 请求封装与 refresh 机制
3. 看板页
4. 行程列表与详情
5. 行程编辑
6. 用户列表与详情
7. 联调与测试
