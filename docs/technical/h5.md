## 1. 文档目标

本文档用于指导 `mn-frontend-h5` 的前端实现与后端对接，覆盖页面结构、路由鉴权、状态管理、接口映射、交互规则和测试要点。

---

## 2. 技术选型

| 项目 | 建议 |
|------|------|
| 前端框架 | React |
| 构建工具 | Vite |
| 路由 | React Router |
| 请求库 | Axios |
| 状态管理 | 轻量方案，推荐 Zustand 或 Context + hooks |
| UI 方案 | 自定义业务组件，移动端优先 |

原则：

- H5 不是后台风格页面，不引入重型桌面组件库
- 尽量以业务组件为主，保证移动端交互体验

## 3. 页面结构


说明：

- 底部固定 Tab：首页 / 我要发布 / 我的

## 4. 路由与鉴权

### 4.1 路由策略

- 游客可访问：首页、行程详情、登录、注册
- 登录后才可访问：发布页、编辑页、个人中心、我的发布、我的收藏

### 4.2 登录回跳

- 未登录访问受保护页面时跳转登录页
- 登录页通过 `redirect` query 保存来源地址
- 登录成功后回跳原页面

### 4.3 Token 处理

- `accessToken` 和 `refreshToken` 存储在 `localStorage`
- 请求拦截器自动携带 access token
- 收到未登录响应时自动尝试 refresh
- refresh 成功后重放原请求
- refresh 失败后清理本地 token 并跳转登录页

## 5. 页面实现与接口映射

### 5.1 首页

职责：

- 展示行程列表
- 提供筛选、分页、下拉刷新、上拉加载

接口：

- `GET /api/v1/trips`

筛选参数：

- `tripType`
- `fromText`
- `toText`
- `datePreset`：`today / tomorrow`
- `pageNum`
- `pageSize`

默认规则：

- 列表按发布时间倒序
- `status=full` 的卡片显示已满并禁用点击

### 5.2 行程详情页

职责：

- 展示完整行程信息
- 根据是否本人展示不同操作栏

接口：

- `GET /api/v1/trips/:id`
- `POST /api/v1/trips/:id/favorite`
- `PATCH /api/v1/trips/:id/status`

操作规则：

- 非本人：收藏 / 联系
- 本人：编辑 / 关闭
- 已满员或已关闭时禁用联系和收藏

### 5.3 发布行程页

接口：

- `POST /api/v1/trips`

表单校验：

- 起点必填
- 终点必填
- 起点和终点不能相同
- 出发日期和时间必填
- 出发时间不能早于当前时间
- 人数范围 1~6
- 价格和面议二选一
- 微信号与手机号至少填写一种
- 备注为选填，用于补充上下车说明、行李要求等信息

成功行为：

- 发布成功后跳转详情页
- Toast 提示“发布成功”

### 5.4 编辑行程页

接口：

- `GET /api/v1/trips/:id`
- `PUT /api/v1/trips/:id`

实现建议：

- 复用发布表单组件
- 页面初始化时加载详情并回填表单
- 只允许编辑本人行程
- 支持编辑并回填备注字段

### 5.5 个人中心页

接口：

- `GET /api/v1/auth/me`
- `PUT /api/v1/users/profile`
- `PUT /api/v1/users/contact`
- `POST /api/v1/users/avatar`

功能：

- 头像展示与更换
- 昵称修改
- 联系方式管理入口
- 我的发布数量角标
- 我的收藏数量角标

### 5.6 我的发布列表

接口：

- `GET /api/v1/me/trips`

规则：

- 按创建时间倒序
- 支持下拉刷新、上拉加载
- 支持关闭行程

### 5.7 我的收藏列表

接口：

- `GET /api/v1/me/favorites`

规则：

- 按收藏时间倒序
- 行程下线后仍展示收藏记录，点击时提示“行程不存在”或“已下线”

### 5.8 登录页

接口：

- `POST /api/v1/auth/login`

成功行为：

- 保存 token
- 回跳 `redirect`

### 5.9 注册页

接口：

- `POST /api/v1/auth/code`
- `POST /api/v1/auth/register`

交互规则：

- 注册验证码请求体带 `type=register`
- 验证码发送后前端进入 60 秒倒计时

### 5.10 重置密码页

接口：

- `POST /api/v1/auth/code`
- `POST /api/v1/auth/password/reset`

交互规则：

- 重置密码验证码请求体带 `type=reset_password`
- 页面默认使用当前登录用户邮箱发码与提交
- 重置密码成功后清理本地登录态并跳转登录页重新登录

交互规则：

- 用户输入邮箱后可点击“发送验证码”
- 验证码输入框为 6 位数字
- 同一邮箱重新发送时，以最新验证码为准
- 点击注册时请求体需包含 `email`、`password`、`code`

成功行为：

- 自动登录
- 保存 token
- 回跳 `redirect`

## 6. 状态管理建议

建议按以下维度拆分状态：

- `auth`：当前用户、token、登录状态
- `tripList`：首页筛选条件、列表数据、分页状态
- `publishForm`：发布/编辑表单数据
- `profile`：个人资料与头像

原则：

- 远程数据优先交给请求层与页面状态管理
- 全局状态只放登录态与少量跨页共享信息

---

## 7. 交互实现规则

### 7.0 UI 变更流程

- 涉及 H5 页面视觉、布局、信息层级调整时，先在仓库根目录 `htmls/*.html` 中制作或修改静态稿
- 先以静态 HTML 验证风格、排版、卡片结构和关键交互态，再同步到 `mn-frontend-h5` 的 React 页面与共享样式
- 当前 v1 视觉基线以 `htmls/v1.html` 为准；后续若有新的 UI 方向，仍先改 HTML 看效果，再进入 H5 实现

### 7.1 Toast

- 统一展示 2 秒
- 成功与错误提示都使用 Toast

### 7.2 弹窗

- 点击蒙层不能关闭
- 必须通过明确按钮关闭

### 7.3 空状态

- 无数据展示插画 + “暂无数据”
- 列表到底展示“没有更多了”

### 7.4 头像上传

- 本地先预览
- 接口失败时回退到原头像

---

## 8. 错误处理

统一错误提示：

- 请求超时：`请求超时，请稍后重试`
- 网络断开：`网络连接已断开，请检查网络`
- 服务端 5xx：`服务器异常，请稍后再试`

业务错误应优先根据 `code` 判断，再使用 `message` 展示。

## 9. 测试用例

### 9.1 认证

- 未登录访问发布页跳转登录
- 登录成功按 `redirect` 回跳
- token 过期后自动 refresh
- refresh 失败后回登录页

### 9.2 首页

- 筛选条件生效
- 下拉刷新成功
- 上拉加载更多成功
- 无数据时展示空状态

### 9.3 发布与编辑

- 起点终点相同校验
- 时间早于当前校验
- 联系方式至少一项校验
- 发布成功跳转详情页
- 编辑页正确回填历史数据

### 9.4 收藏与个人中心

- 未登录点击收藏跳转登录
- 已登录收藏 toggle 正确切换
- 头像上传失败后预览回退
- 联系方式修改成功后重新获取个人信息

---

## 10. 开发建议

建议实现顺序：

1. 路由、基础布局、请求封装
2. 登录注册与 token 刷新
3. 首页与详情页
4. 发布页与编辑页
5. 个人中心
6. 我的发布与我的收藏
7. 头像上传与收尾联调
